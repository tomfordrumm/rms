# Phase 1: Создание ресторана и базовые настройки

## Суть задачи
Реализовать в админке полноценный функционал первичного заполнения и редактирования ресторана с базовыми настройками:
- name, description
- slug (для публичных страниц, с авто-генерацией и ручной правкой)
- logo и cover
- контакты (в удобном формате)
- часы работы по дням недели (минимум: `open_time` + `close_time`)
- перерывы/выходные (опционально)
- закрытия на конкретные даты (`closed_dates`)

Ключевой сценарий:
- отдельной страницы “Создание ресторана” нет;
- после регистрации пользователь сразу попадает на страницу “Настройки ресторана”;
- пока ресторан не создан (не заполнены обязательные поля), доступ к остальной системе заблокирован;
- после первичного сохранения та же страница используется для дальнейшего редактирования.
